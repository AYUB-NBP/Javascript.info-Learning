Function expressions

Since functions are values. They can get passed to a variable within any expression.
And that's how you create a Function expression.

The syntax for a Function Expression is as follows:

let functionName = function(Parameters) {//Code};

Example:

let hello = function() {alert('Hello.');};
hello();  

This is different from the Function Declaration used before now that goes like this:

function functionName(Parameters) {
//code body}

Example:

function helloD(){alert('Hello!')};  

In both cases the function is basically source code that gets stored like any value into a variable;
which in this context is whatever name the function happens to be given.

Function is a value

Since a function is a value, we can print it out.

function sayBye() {
alert('Bye!') } 

As long as there isn't a double parentheses() after a declared function, it will not be called.

The code above shows its string representation, which is the source code.


We can copy a function to another variable like any value:

function func_1() { alert('Writing...'); };
let func_2 = func_1;

Function Expression vs Function Declaration

  • Functions are values. They can be assigned, copied or declared in any place of the code.
  • If the function is declared as a separate statement in the main code flow, that’s called a “Function Declaration”.
  • If the function is created as a part of an expression, it’s called a “Function Expression”.
  • Function Declarations are processed before the code block is executed. They are visible everywhere in the block.
  • Function Expressions are created when the execution flow reaches them.
